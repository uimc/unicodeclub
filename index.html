Introduction

With the exception of the information emoji (‚Ñπ), the trademark emoji (‚Ñ¢Ô∏è) and the "m" emoji (‚ìÇÔ∏è), for an emoji to work as a domain name, it must be converted into "punycode". Punycode is a character encoding method used for internationalized domain names (IDNs). This representation is used when registering domains containing special characters. The ASCII representation starts with the prefix "xn--" and is followed by the emoji-containing domain name encoded as punycode. For example, "xn--qei" is "‚ù§" when converted back to Unicode.

Each emoji has a unique punycode representation. For example, "üòâ" is represented as "xn--n28h". There are several generators on the Internet that allow one to convert emoji to punycode and back, but the most popular is punycoder.com

Notable Events

Prior to April 2001, domain names were limited to 37 characters: 26 English letters, 0 to 9, and the hyphen (-).

On 19 April 2001, ICANN and VeriSign expanded the allowed set of characters to include a limited range of symbols drawn from a few different Unicode character sets below:

At least four single-character unicode domains were created on this day: ‚òª.com (xn--84h.com), ‚ô®Ô∏è.com (xn--j6h.com), ‚ô®Ô∏è.net (xn--j6h.net), ‚òÆÔ∏è.com (xn--v4h.com).

In 2003, i‚ô•you.com (xn--iyou-5u3b.com), i‚ô•.com (xn--i-n3p.com), and over a thousand other heart-related unicode domains were registered by Menno de Ruiter for the first mass-scale emoji project on the internet in conjunction with the two principal programmers for the punycode/IDN project at VeriSign and Galcomm ICANN registrar. VeriSign would later delete the majority of these heart-related unicode domains, without notice, leaving the project owner out of hundreds of thousands of dollars.

In 2005, ICANN deleted all .org domain names that contained unicode symbols without notice, leaving owners confused, upset, and empty-handed.

In 2008, the IDNA2008 standard is initiated by ICANN and advises against emoji domains due to security and usability concerns.

In 2009, ICANN introduced Internationalized Domain Names (IDNs), allowing domain names in non-Latin scripts to be registered.

In 2010, the Unicode Consortium formally recognized 722 emojis and adopted 33 of these unicode symbols as emojis: www.unicode.org/reports/tr51/.

The IDNA2008 is published and accepted by ICANN, effectively banning any further emoji-based .com and .net registrations: www.unicode.org/reports/tr46/.

Emoji Domains Today

As of May 2025, only country-code top-level domains (ccTLDs) are able to allow the registration of unicode/emoji domains. Of the 308 ccTLDs in existence today, only 6 allow for emoji domain registrations:

.fm .kz .st .to .uz .ws

Note: The registration of an emoji domain can be more difficult since it is sometimes not possible to enter emoji characters into the online registration fields at some domain name registrars. Try using the punycode version of the unicode/emoji instead.

The availability of an emoji domain name can be verified by checking the WHOIS data of the punycode version of the domain.
